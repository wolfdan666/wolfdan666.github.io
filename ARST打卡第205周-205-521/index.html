迁移基于 LevelDB 1.18 增加了备份功能的 LevelDB 数据库到 RocksDB 7.0，需要注意以下几点：

备份数据的存储位置不同：在 LevelDB 中，备份数据的存储位置由用户指定，而在 RocksDB 中，备份数据的存储位置是 RocksDB 的默认目录，通常是在 rocksdb 数据库所在的同级目录下的 db_backup 目录中。因此，在迁移过程中，需要根据具体情况来确定备份数据的存储位置。

增加了新的配置选项：在 RocksDB 中，备份相关的配置选项与 LevelDB 不同，需要根据具体需求进行调整。比如，RocksDB 中提供了两种备份方式：单线程备份和多线程备份，可以根据需要进行选择。同时，RocksDB 还提供了多种备份格式，包括 SST 文件备份和 Checkpoint 备份，可以根据具体需求来选择合适的备份格式。

数据库的 API 和接口不同：在 RocksDB 中，部分 API 和接口与 LevelDB 不同，需要进行相应的修改。比如，RocksDB 中没有 LevelDB 中的 WriteBatchWithIndex 接口，而是提供了更加灵活和高效的 WriteBatchWithContext 接口。因此，在迁移过程中，需要根据具体情况来修改代码，以保证代码的兼容性和正确性。

RocksDB 支持更多的配置选项和特性：RocksDB 相较于 LevelDB，提供了更多的配置选项和特性，比如支持多种不同的压缩算法、多种数据存储格式等。因此，在迁移过程中，需要根据具体需求来进行相应的配置，以获得更好的性能和效果。

综上所述，在迁移基于 LevelDB 1.18 增加了备份功能的 LevelDB 数据库到 RocksDB 7.0 时，需要注意以上几点，并根据具体情况进行相应的调整和修改，以保证迁移的顺利和正确性。
